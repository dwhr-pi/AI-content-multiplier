# Module in OpenClaw nutzen

Die Modulordner in diesem Starter stammen aus den vorhandenen Paketen und
wurden hier entpackt, damit sie lokal und ohne ZIP-Hin-und-Her nutzbar sind.

## Empfohlene Reihenfolge

1. `absicherung/`
2. `kontext/`
3. `daten/`
4. `intelligenz/`
5. `boardroom_openclaw/`

## Umgang mit Claude-Hinweisen

Wenn in einzelnen Moduldateien noch von Claude gesprochen wird:

- lies die fachliche Absicht, nicht nur den exakten Wortlaut
- nutze OpenClaw als Arbeitsagenten
- uebernimm Inhalte, Struktur und Fragen
- ignoriere nur die Claude-spezifische Bedienoberflaeche

## Ziel

Die Module bleiben inhaltlich erhalten, werden aber in einem OpenClaw-freundlichen
Arbeitsrahmen genutzt.
